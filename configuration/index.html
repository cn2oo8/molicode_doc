# 配置说明

以下配置将介绍如何使用本工具生成业务代码。配置主要分为两部分信息。

# 1. 全局配置（不使用maven仓库请略过本说明）

主要是配置maven仓库相关信息。因为本工具主要依赖maven仓库共享代码模板，故而需要配置maven相关运行环境。 如果您已经在环境变量中配置了maven运行的相关信息，本部分可以忽略。

（不使用maven仓库请忽略，该功能已废弃）

# 2. 项目配置

因为我们针对不同的项目有不同的配置，如模板不一样，工程的包路径等也不一样。故而这里有一个项目这样一个概念。 项目的概念便于您在不同项目之间自由切换，同时保存历史配置信息，当您需要回过头去开发之前的工程的时候，只需要切换回去即可直接开发。

# 项目列表

如【图2-1】所示，项目列表可以进行项目分页查看，项目新建，删除，修改等基础功能。同时项目可以进行复制，设置为默认项目等操作。默认项目即工作项目。

# 数据库设置

如【图2-2】所示，数据库设置主要是数据的链接信息和用户名密码等。目前只支持MySQL数据库，后期如有其它数据库请联系沟通。

# 工程目录设置

如【图2-3】所示，工程目录设置主要是一些工程目录相关的配置信息。

【图2-3】 工程目录设置

字段说明

  • tableModel 存放目录： tableModel为数据库模型的XML中间文件，生成代码前有多此一举的一个中间文件。主要方便在生成代码前可以进行微调和自定义配置；
  • 代码输出根目录： 如果首次使用，建议找个临时文件夹输出测试一下。如果比较熟练，有充分的信任，可以直接设置为工程根目录；
  • 模板类型：可选为本地模板或者maven， 其中本地模板主要是开发模板使用，正常情况下均使用maven模式；
  • 模板根目录： 只有模板类型为本地模板才需要设置，指向本地开发模板的根目录；即您的auto
  • maven 功能已废弃，请忽略。

# 业务代码设置

如【图2-4】所示，业务代码设置主要设置生成代码相关的一些业务相关信息。

【图2-4】 业务代码设置

字段说明：

  • author（作者）：可以通过 ${config.author} 获取，一般用来指明代码作者；
  • basePackage(基础包路径): 可以通过 ${config.basePackage} 获取, 主要决定代码的输出目录。
  • artifactId（maven项目前缀）: 可通过 ${config.artifactId} 获取，一般内容比较使用，主要是用于模块文件的前缀； 如 b2b-user 则一般生成的maven模块可能有: b2b-user-common, b2b-user-dao等。
  • category(应用子包名): 可通过 ${config.category} 获取，用于类的子分包；
